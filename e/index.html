What is OpenToonz?

  • It is a 2D animation software.

    Based on the software "Toonz", which was originally developed by Digital Video S.p.A. in Italy, it was customized by the world-famous Studio Ghibli, and used in creating many of their works. Dwango has now launched the open-source OpenToonz project, in cooperation with Digital Video and Studio Ghibli, to make this powerful production tool more widely available.

  • It can be used by anyone, free of charge, no matter whether the purpose of use is commercial or not.
  • It is available as open-source software, so that users can modify its source code freely.
  • We aim to develop a new platform for connecting academic research to the front line of animation production.

Characteristics of OpenToonz

How it has been used in Studio Ghibli

Since the first use of OpenToonz's predecessor, "Toonz", to realize some cuts of Princess Mononoke, the software has been employed in the ink and paint, color design, and digital composition processes* of almost all works at Studio Ghibli. After "Arrietty/The Secret World of Arrietty", it was customized in-house, and evolved to become a better fit for their production style.

* Ink and paint, color design, and digital composition: Processes involved from the scanning of hand-drawn pictures to the production of final footage.

Open source, which can be used by anyone free of charge

The source code of OpenToonz can be modified freely, in accordance with its license*. It can be used by anyone, free of charge, no matter whether the purpose of use is commercial or not. It can be used in all kinds of fields, such as professional or amateur production, as well as research, and education.

* For further information on the license of the software (installer), see the rules of use of OpenToonz. Its source code is available under the terms of the New BSD License.

Functions to meet on-site needs

OpenToonz is equipped with unique functions that have been improved iteratively based on the feedback of production staff. This includes the scanning tool "GTS", which is specialized in animation production, and was developed at Studio Ghibli. For details, see "Introduction of the functions unique to OpenToonz" below.

Effect development kit (SDK)

Dwango has freshly developed a SDK for image processing and effects plugins for OpenToonz. By using the SDK, anyone can add new effects to OpenToonz. Video expression researchers can receive swift feedback from on-site workers and the community by developing and releasing their results as plugins.

The effects developed by Dwango are also released

By using the plugin SDK, the effects developed by Dwango's machine leaning research and development team are now available in OpenToonz. They include effects such as automatically changing picture styles by applying deep learning technology, and producing affected incident light as seen in classic works from before the digitization of the production environment.

Introduction of the functions unique to OpenToonz

  • GTS, a scanning tool developed by Studio Ghibli

    • Sequence number scanning can be conducted efficiently according to in-between animation numbers.
    • Compatible with 4 types of scanning
      (black and white, colored, with or without binarization).
    • The settings for scanning can be saved. Processes can be reproduced when some cuts are rescanned.
    • Compatible with the TWAIN standard.

  • Digital painting

    • Anti-aliased lines can be used.
    • Since the data of paintings are independent of those of the palette, it is easy to paint with provisional colors.
    • Smooth operation even with high-definition image data.

  • Filming

    • Equipped with a timesheet-type (x-sheet) interface.
      (There is an experimental timeline mode available.)
    • Effects can be synthesized easily with the GUI using the node tree.
    • Images can be handled with real scale considering resolution [dpi] (units, such as millimeter, are specified).
    • It is possible to add more effects by using plugins.

  • Other features

    • Compatible with both raster and vector images.
    • Equipped with a function for the interconversion between sequence images of colored binarized TGAs and the OpenToonz format, to secure compatibility with existing workflows.
    • A Japanese localized version is available, as well as community-maintained translations in German, Spanish, French, Italian, Russian, and Chinese.

FAQ

Please tell me the recommended system requirements.

The system requirements for using OpenToonz comfortably are as follows:

Windows

  • Microsoft Windows 7 SP1/8.1/10 (64bit required)
  • Intel Core i series processor
  • 4GB or more RAM
  • 500 MB or more available storage space
  • 1280 x 1024 or higher monitor resolution

OS X

  • OS X 10.9 or a later version
  • Intel Core i series processor
  • 4 GB or more RAM
  • 500 MB or more available storage space
  • 1280 x 1024 or higher monitor resolution

The system requirements for using GTS comfortably are as follows:

  • Microsoft Windows 7/8.1/10
  • Intel Core i series processor
  • 2 GB or more RAM
  • 8 MB or more available storage space
  • Scanner compatible with the TWAIN standard (including the driver software).
    ※ The EPSON DS-50000 is recommended.
If I find a defect in the software or have a problem, what should I do?
OpenToonz has a Google group to provide a place where users can share and solve troubles. Join the group to request functional improvements, explore workflows and current features, and have discussions with other users.
In addition, there is a forum for developers and video researchers on GitHub. Please only post development-related topics there, such as substantiated and reproducible bug reports. Feature requests belong on the Google group.
